Integration Guide

How to use in your Django templates:

  1. Include the JavaScript file: Add the script tag to your template
  2. Add the search HTML: Include the search input and results container
  3. Add data attributes: Ensure your photo items have the required data attributes
  4. Initialize the component: Create a new PhotoSearch instance

1. HTML Structure Required:

<!-- Photo Search Bar --> <div class="photo-search-container"> <input type="text" class="photo-search-input" placeholder="Search photos..."> <div class="photo-search-results"></div> </div> <!-- Photo Grid with data attributes --> <div class="photo-grid"> <div class="photo-item" data-photo-id="{{ photo.id }}" data-description="{{ photo.description }}" data-tags="{{ photo.tags }}" data-date="{{ photo.date }}"> <!-- photo content --> </div> </div>

2. JavaScript Initialization:

// Basic initialization const photoSearch = new PhotoSearch(); // With custom options const photoSearch = new PhotoSearch({ searchInputSelector: '.my-search-input', resultsContainerSelector: '.my-results', photoGridSelector: '.my-photo-grid', searchDelay: 500, minSearchLength: 3 });

3. Available Options:

{ searchInputSelector: '.photo-search-input', // Search input selector resultsContainerSelector: '.photo-search-results', // Results container selector photoGridSelector: '.photo-grid', // Photo grid selector searchDelay: 300, // Debounce delay in ms minSearchLength: 2, // Minimum characters to search highlightClass: 'search-highlight', // CSS class for highlighted terms noResultsClass: 'no-search-results', // CSS class for no results loadingClass: 'search-loading' // CSS class for loading state }

4. Public Methods:

photoSearch.refresh(); // Refresh photos collection photoSearch.clearSearch(); // Clear current search photoSearch.destroy(); // Clean up event listeners

Features

  • Real-time search: Search as you type with debouncing
  • Multi-field search: Searches title, description, tags, and date
  • Highlighted results: Search terms are highlighted in results
  • Responsive design: Uses CSS variables from your base.scss
  • Keyboard navigation: Escape key to clear search
  • Click to scroll: Click results to scroll to photos
  • Statistics: Shows count of filtered vs total photos
  • Easy integration: Drop-in component for any template

CSS Variables Used

The component automatically uses your base.scss CSS variables:

--text-default // Default text color --background-default // Default background color --primary-default // Primary color for highlights --text-100, --text-200 // Text color variations --background-100 // Background color variations
