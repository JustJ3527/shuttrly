Integration Guide

How to implement HTMX + PhotoSearch in your Django app:

  1. Include HTMX: Add HTMX to your base template
  2. Add PhotoSearch: Include the PhotoSearch JavaScript component
  3. Set up HTMX attributes: Configure HTMX for dynamic content loading
  4. Handle HTMX events: Re-initialize PhotoSearch after content updates

1. Include HTMX in your base template:

<!-- In your base.html --> <script src="https://unpkg.com/htmx.org@1.9.10"></script>

2. Add HTMX attributes to your post type options:

<div class="post-type-option" hx-get="{% url 'posts:post_create' %}?type=single_photo" hx-target="#post-content-area" hx-push-url="true" hx-swap="innerHTML" hx-indicator="#loading-indicator"> <!-- content --> </div>

3. Set up the content area and loading indicator:

<!-- Loading Indicator --> <div id="loading-indicator" class="htmx-indicator"> <div class="spinner-border text-primary" role="status"> <span class="visually-hidden">Loading...</span> </div> </div> <!-- Content Area --> <div id="post-content-area"> <!-- This will be updated by HTMX --> </div>

4. Handle HTMX events to re-initialize PhotoSearch:

// Re-initialize search after HTMX content updates document.body.addEventListener('htmx:afterSwap', function(event) { if (event.target.id === 'post-content-area') { setTimeout(() => { initializePhotoSearch(); }, 100); } });

5. Modify your Django view to handle HTMX requests:

def get_template_names(self): """Return appropriate template based on request type""" if self.request.headers.get('HX-Request'): return ["posts/partials/post_content.html"] return [self.template_name]

Benefits of HTMX + PhotoSearch Integration

  • No page reloads: Smooth transitions between post types
  • Maintained state: Form data and selections are preserved
  • Better UX: Faster navigation and smoother interactions
  • SEO friendly: URLs are updated with hx-push-url
  • Progressive enhancement: Works without JavaScript for basic functionality
  • Real-time search: PhotoSearch component works seamlessly with HTMX

HTMX Attributes Used

hx-get // URL to fetch content from hx-target // Element to update with response hx-push-url // Update browser URL hx-swap // How to insert new content hx-indicator // Element to show during request
